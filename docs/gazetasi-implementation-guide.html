Si Parashikime

Udhezues implementimi per Gazeta Si

Ky dokument eshte pergatitur si handoff praktik per ekipin teknik te Gazeta Si: si te marrin kodin, si te lidhen me databazen e tyre, si te bejne deploy, dhe si te arrijne nje eksperience me te njejtin account mes WordPress dhe prediction market.

1. Si te ndahet repo-ja me IT

Repo-ja aktuale eshte:

https://github.com/po0xtirana/si-parashikime
  1. Hape repo-n ne GitHub.
  2. Shko te Settings -> Collaborators and teams.
  3. Shto personin e IT me username ose email.
  4. Jepi akses Write ose Admin.
Rekomandim Per handoff me afat me te gjate, repo-ja duhet transferuar ne nje account ose organizate te Gazeta Si, ne menyre qe pronari teknik te jete institucioni dhe jo nje profil personal.

2. Stack-u aktual

  • React + Vite + TypeScript
  • Tailwind + shadcn/ui
  • Supabase per databaze, auth, RLS dhe SQL functions

3. Si ndryshohen databaza dhe kredencialet

3.1 Variablat e ambientit

Kopjoni .env.example ne .env dhe plotesoni:

VITE_SUPABASE_URL="https://YOUR_PROJECT_REF.supabase.co"
VITE_SUPABASE_PUBLISHABLE_KEY="YOUR_PUBLISHABLE_KEY"
Kujdes Mos vendosni asnjehere sb_secret_* ose service_role ne skedaret VITE_*. Cdo vlere me VITE_ ekspozohet ne frontend.

3.2 Supabase CLI

Ne supabase/config.toml vendosni project ref-in real:

project_id = "your-supabase-project-ref"

Pastaj ekzekutoni:

npx supabase login
npx supabase link --project-ref YOUR_PROJECT_REF
npx supabase db push

3.3 Rigjenerimi i tipeve

npx supabase gen types typescript --linked --schema public > src/integrations/supabase/types.ts

5. Rolet

Aktualisht roli admin kontrollohet nga tabela user_roles.

insert into public.user_roles (user_id, role)
values ('SUPABASE_AUTH_USER_ID', 'admin')
on conflict do nothing;

Rekomandohet qe Gazeta Si te shtoje edhe nje rol editor, qe te krijoje dhe mbylle tregje pa pasur akses te plote administrativ.

7. Si behen te njejtat llogari ne WordPress dhe market

Synimi eshte qe nje perdorues i loguar ne www.gazetasi.al te hape market.gazetasi.al pa u loguar perseri.

Qasja e duhur Mos u perpiqni te ndani cookie-t e WordPress direkt me aplikacionin React. Rruga e shendetshme eshte OAuth / OpenID Connect, me WordPress si Identity Provider.

7.1 Rrjedha e rekomanduar

  1. WordPress ben autentikimin kryesor.
  2. Prediction market ofron butonin Hyr me Gazeta Si.
  3. Perdoruesi ridrejtohet te WordPress me Authorization Code + PKCE.
  4. Nese ai eshte tashme i loguar ne Gazeta Si, ridrejtimi mbyllet menjehere.
  5. Market app krijon sesionin e vet dhe perdoruesi hyn pa friction.

7.2 Cfare duhet ne WordPress

  • plugin OAuth/OIDC server
  • endpoints per authorize, token dhe idealisht userinfo
  • client_id, redirect_uri dhe scopes

7.3 Cfare duhet ne prediction market

  • faqe login me butonin Hyr me Gazeta Si
  • route callback si /auth/callback
  • mapim i identitetit WordPress me user-in e market-it
  • krijim automatik i profilit ne hyrjen e pare

7.4 Tabele e rekomanduar per mapim identitetesh

create table if not exists public.external_accounts (
  id uuid primary key default gen_random_uuid(),
  provider text not null,
  external_user_id text not null,
  user_id uuid not null references auth.users(id) on delete cascade,
  created_at timestamptz not null default now(),
  unique (provider, external_user_id)
);

8. Faza e implementimit

Faza 1: Marrja ne dore e projektit

  1. Klononi repo-n.
  2. Krijoni .env nga .env.example.
  3. Krijoni nje projekt te ri Supabase.
  4. Lidhni projektin dhe aplikoni migrimet.
  5. Rigjeneroni types.ts.
  6. Nisni aplikacionin me npm run dev.

Faza 2: Integrimi ne infrastrukturen e Gazeta Si

  1. Deploy ne market.gazetasi.al.
  2. Vendosja e env vars ne ambientin e deploy-it.
  3. Krijimi i nje admini fillestar.
  4. Verifikimi i panelit /admin.

Faza 3: SSO me WordPress

  1. Zgjedhja e plugin-it OAuth/OIDC ne WordPress.
  2. Krijimi i klientit per market-in.
  3. Shtimi i callback route ne aplikacion.
  4. Krijimi i tabeles external_accounts.
  5. Ndryshimi i UX se login-it qe te kaloje nga Gazeta Si.
