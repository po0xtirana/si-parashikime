Si Parashikime

Udhezues implementimi

Ky dokument tregon cfare duhet bere realisht per ta vene projektin ne pune: si te lidhni databazen, si te jepni admin access, si te beni deploy, dhe si te arrini login me te njejtin account mes WordPress dhe market-it.

1. Qellimi

Qellimi eshte qe projekti i prediction market te jeton ne market.gazetasi.al, ndersa faqja kryesore te mbetet ne www.gazetasi.al, me te njejtin account per perdoruesit.

2. Cfare ju duhet para se te filloni

  1. Akses ne repo: https://github.com/po0xtirana/si-parashikime
  2. Nje projekt i ri ne Supabase
  3. Nje subdomain si market.gazetasi.al
  4. Akses ne WordPress
  5. Nje plugin OAuth ose OpenID Connect server ne WordPress per SSO

3. Skedaret qe duhen ndryshuar

  • .env
  • supabase/config.toml
  • src/integrations/supabase/types.ts

Nese do implementoni SSO me WordPress, do preken edhe:

  • src/pages/Auth.tsx
  • src/hooks/useAuth.tsx
  • src/App.tsx

4. Nisja lokale

git clone https://github.com/po0xtirana/si-parashikime.git
cd si-parashikime
npm install
npm run dev

5. Krijimi i Supabase-it te ri

Ne Supabase krijoni nje projekt te ri dhe merrni keto vlera:

  • Project URL
  • Publishable key
  • Project ref
Mos e beni Mos vendosni ne frontend sb_secret_* ose service_role. Keto mbahen vetem ne backend.

6. Mbushja e `.env`

Kopjoni .env.example ne .env dhe vendosni:

VITE_SUPABASE_URL="https://YOUR_PROJECT_REF.supabase.co"
VITE_SUPABASE_PUBLISHABLE_KEY="YOUR_PUBLISHABLE_KEY"

7. Ndryshimi i `supabase/config.toml`

project_id = "REAL_PROJECT_REF"

8. Aplikimi i databazes

npx supabase login
npx supabase link --project-ref YOUR_PROJECT_REF
npx supabase db push

Kjo krijon tabelat, funksionet dhe migrimet qe jane ne supabase/migrations/.

9. Rigjenerimi i tipeve

npx supabase gen types typescript --linked --schema public > src/integrations/supabase/types.ts

Tabelat kryesore

  • profiles
  • user_roles
  • markets
  • bets
  • price_history

Kolonat e rendesishme

  • markets.article_url
  • bets.is_public

10. Si krijohet admini i pare

  1. Krijoni nje account normal ne website.
  2. Hapni Supabase SQL Editor.
  3. Ekzekutoni:
select id, email
from auth.users
order by created_at desc;

Merrni id e user-it dhe jepini rolin:

insert into public.user_roles (user_id, role)
values ('USER_ID_KETU', 'admin')
on conflict do nothing;

11. Si perdoret paneli admin

Paneli ndodhet te /admin. Nga aty mund te krijoni tregje dhe te vendosni:

  • titull
  • pershkrim
  • kategori
  • date mbylljeje
  • vlera fillestare Po / Jo
  • image URL
  • article URL

12. Deploy

Rekomandohet:

  • www.gazetasi.al per WordPress
  • market.gazetasi.al per prediction market

Settings tipike te deploy-it:

  • Build command: npm run build
  • Output directory: dist

13. Si behet login me te njejtin account

Synimi eshte qe nje user i loguar ne www.gazetasi.al te hyje ne market.gazetasi.al pa login te dyte.

Rruga e duhur Perdorni WordPress si burim identiteti dhe OAuth / OpenID Connect per login. Mos ndani cookie-t e WordPress direkt me aplikacionin React.

14. Cfare duhet bere ne WordPress

  1. Instaloni nje plugin qe e ben WordPress OAuth/OIDC server.
  2. Krijoni nje client per market-in.
  3. Vendosni redirect URI:
https://market.gazetasi.al/auth/callback

Pastaj ruani keto vlera:

  • client_id
  • client_secret
  • issuer
  • authorize_url
  • token_url
  • userinfo_url

Scopes minimale:

openid email profile

15. Cfare duhet bere ne aplikacion

  1. Hiqni login-in klasik nga UI ose fsheheni.
  2. Shtoni butonin Hyr me Gazeta Si.
  3. Shtoni route /auth/callback.
  4. Ne callback merrni code nga WordPress.
  5. Shkembeni code me token.
  6. Merrni profilin e user-it.
  7. Gjeni ose krijoni user-in perkates ne Supabase.
  8. Krijoni sesionin ne market.

16. Tabela e mapimit mes WordPress dhe Supabase

create table if not exists public.external_accounts (
  id uuid primary key default gen_random_uuid(),
  provider text not null,
  external_user_id text not null,
  user_id uuid not null references auth.users(id) on delete cascade,
  created_at timestamptz not null default now(),
  unique (provider, external_user_id)
);

Kjo lidh user-in e WordPress me user-in e market-it.

17. Backend i nevojshem per SSO

Per SSO duhet nje endpoint backend, p.sh.:

https://market.gazetasi.al/api/auth/wordpress/callback

Ky endpoint ben:

  1. code -> token
  2. merr profilin e user-it nga WordPress
  3. gjen ose krijon user ne Supabase
  4. ruan mapimin ne external_accounts
  5. kthen user-in ne aplikacion me sesion aktiv
Sekret backend Per kete hap do duhet SUPABASE_SERVICE_ROLE_KEY. Ky key nuk futet kurre ne frontend.

18. Skedaret qe pritet te preken per SSO
